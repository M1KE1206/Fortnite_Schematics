# Real Names + Auto Icons Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Geïmporteerde schematics krijgen hun echte in-game naam (vertaaltabel uit de stw-planner-sitemap, meegecommit) en automatisch een wiki-icon; slot 6 heet voortaan "Perk 6".

**Architecture:** Eenmalig generator-script schrijft `server/data/schematicNames.ts`; `parseCampaignSchematics` doet een basiskey-lookup met prettify-fallback (testbaar via geïnjecteerde map). Client: `findIconUrl` helper in `wiki.ts`, `mutate` (functionele update) in AppStateContext, en een achtergrondloop in SchematicsSection die na import per schematic de icon ophaalt.

**Tech Stack:** Bestaand project; geen nieuwe dependencies.

## Global Constraints

- Geen emojis; iconen alleen lucide-react; UI Engels; server/-imports met `.js`-extensie.
- Runtime praat NOOIT met stw-planner (alleen het eenmalige script doet dat; gegenereerd bestand wordt gecommit).
- VEILIGHEID: roep NOOIT `/api/stw-sync/sync` of `/schematics` aan tegen de dev-server (echte device auth op deze machine). Het generator-script (stw-planner sitemap, publiek en read-only) mag WEL gedraaid worden.
- `findIconUrl` gooit nooit; faalt stil met `null`.
- Bestaande parse-tests blijven deterministisch: alle bestaande `parseCampaignSchematics`-aanroepen in tests krijgen expliciet `{}` als names-parameter.

`scripts/generate-schematic-names.mjs`:
```js
// Eenmalig draaien: node scripts/generate-schematic-names.mjs
import { mkdirSync, writeFileSync } from 'node:fs';

const SITEMAP = 'https://stw-planner.com/sitemap.xml';
const RARITY = new Set(['c', 'uc', 'r', 'vr', 'sr']);

const xml = await (await fetch(SITEMAP)).text();
const re = /schematics\/([a-z0-9-]+)\/([A-Za-z0-9_]+)/g;
const map = {};
let skipped = 0;
let m;
while ((m = re.exec(xml)) !== null) {
  const [, nameSlug, sid] = m;
  const sidTokens = sid.toLowerCase().split('_');
  if (RARITY.has(sidTokens[sidTokens.length - 1])) sidTokens.pop();
  const key = sidTokens.join('_');
  const nameTokens = nameSlug.split('-');
  if (RARITY.has(nameTokens[0])) nameTokens.shift();
  const name = nameTokens.map((w) => (w ? w[0].toUpperCase() + w.slice(1) : w)).join(' ');
  if (map[key] && map[key] !== name) {
    skipped++;
    continue;
  }
  map[key] = name;
}

const sorted = Object.fromEntries(Object.entries(map).sort(([a], [b]) => a.localeCompare(b)));
const body = `// Generated by scripts/generate-schematic-names.mjs - do not edit by hand.\nexport const SCHEMATIC_NAMES: Record<string, string> = ${JSON.stringify(sorted, null, 2)};\n`;
mkdirSync('server/data', { recursive: true });
writeFileSync('server/data/schematicNames.ts', body);
console.log(`Wrote ${Object.keys(sorted).length} names (${skipped} conflicting duplicates skipped).`);
```

Run: `node scripts/generate-schematic-names.mjs` — Expected: "Wrote <honderden> names". Controleer steekproefsgewijs 2 entries in het gegenereerde bestand (bv. een bekende naam als Swan of Siegebreaker aanwezig).

export async function findIconUrl(name: string): Promise<string | null> {
  try {
    const results = await searchWikiIcons(`${name} schematic`);
    const hit = results.find((r) => r.thumbnailUrl);
    if (!hit?.thumbnailUrl) return null;
    return await toDataUrl(hit.thumbnailUrl);
  } catch {
    return null;
  }
}

  function handleImported(created: Schematic[]) {
    update({ schematics: [...state.schematics, ...created] });
    void fetchIconsFor(created);
  }

  async function fetchIconsFor(created: Schematic[]) {
    for (const item of created) {
      const icon = await findIconUrl(item.name);
      if (!icon) continue;
      mutate((s) => ({
        ...s,
        icons: { ...s.icons, [item.name.toLowerCase()]: icon },
        schematics: s.schematics.map((x) => (x.id === item.id ? { ...x, iconUrl: icon } : x)),
      }));
    }
  }
